# 📱 OpenMobileProxy

[![Node.js](https://img.shields.io/badge/Node.js-v18%2B-green.svg)](https://nodejs.org)
[![Kotlin](https://img.shields.io/badge/Kotlin-Android-blue.svg)](https://kotlinlang.org)
[![React](https://img.shields.io/badge/React-v18-cyan.svg)](https://react.dev)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

**OpenMobileProxy** — это полноценная открытая (Open-Source) экосистема для создания собственных приватных мобильных прокси-ферм из обычных смартфонов Android и любого виртуального сервера (VPS).

Проект решает задачу получения чистых, динамических мобильных IP-адресов сотовых операторов (MegaFon, Beeline, MTS, Tele2, Yota и др.) без необходимости покупать дорогостоящие коммерческие прокси-подписки.

## 💡 О проекте и Архитектура

Система состоит из двух ключевых компонентов:
1. **Серверная часть (Node.js VPS + React Web Dashboard)**:
   - Слушает входящие соединения от ваших скриптов/софта по протоколу SOCKS5 или HTTP CONNECT.
   - Мультиплексирует бинарный сетевой трафик через устойчивое WebSocket-соединение с телефоном.
   - Предоставляет современную веб-панель управления с мониторингом устройств, замером скорости, просмотром логов в реальном времени и управлением паролями.

2. **Мобильный клиент (Android App на Kotlin)**:
   - Работает в фоновом режиме (`ForegroundService`) с постоянным уведомлением и `WakeLock` (не уходит в сон).
   - Выполняет роль прозрачного ретранслятора сетевых пакетов.
   - Взаимодействует со `Службой специальных возможностей` (Accessibility Service) для автоматического переключения режима «В самолете» (Airplane Mode) ради смены IP-адреса сотовой вышки.

```
┌────────────────────────────────┐         WebSocket (Encrypted)         ┌────────────────────────────────┐
│      Android Мобильный Клиент  │ ◄───────────────────────────────────► │      VPS Сервер Node.js        │
│  (SOCKS5 Relay + Accessibility)│                                       │ (TCP Proxy Server + Web Panel) │
└────────────────────────────────┘                                       └────────────────────────────────┘
                                                                                         ▲
                                                                                         │ SOCKS5 (port 10001+)
                                                                                         ▼
                                                                         ┌────────────────────────────────┐
                                                                         │       Клиентский Софт / Скрипт │
                                                                         └────────────────────────────────┘
```

## 🛠 Инструкция по установке

### Этап 1: Установка сервера (VPS)

Для работы сервера требуется любой виртуальный сервер (VPS/VDS) под управлением Linux (Ubuntu 20.04+, Debian 11+ или AlmaLinux).

---

#### а) Автоматический деплой через SSH (Рекомендуется)

Этот способ автоматически зальет код на ваш VPS, настроит фаервол (UFW), установит все зависимости и запустит процесс под фоновым менеджером `PM2`.

1. Клонируйте репозиторий на ваш локальный компьютер:
   ```bash
   git clone https://github.com/makssest06-alt/open-mobile-proxy.git
   cd open-mobile-proxy
   ```
2. Скопируйте шаблон конфигурации `.env.example` в `.env`:
   ```bash
   cp .env.example .env
   ```
3. Откройте `.env` и укажите данные вашего VPS:
   ```env
   VPS_HOST=123.45.67.89        # IP адрес вашего VPS
   VPS_PORT=22                 # SSH порт
   VPS_USER=root               # Пользователь SSH
   VPS_PASS=your_vps_password  # Пароль от VPS
   ```
4. Запустите команда автоматического деплоя:
   ```bash
   npx tsx deploy.ts
   ```
После завершения панель управления будет доступна по адресу `http://ИП_ВАШЕГО_VPS:3000`.

---

#### б) Ручной запуск сервера (Manual / Node.js)

Если вы хотите запустить сервер вручную прямо на локальной машине или настроить VPS самостоятельно:

1. Убедитесь, что установлен **Node.js v18+**.
2. Установите зависимости:
   ```bash
   npm install
   ```
3. Создайте файл `.env` на основе примера:
   ```bash
   cp .env.example .env
   ```
4. Скомпилируйте и запустите сервер:
   ```bash
   # Режим разработки
   npm run dev

   # Продакшн сборка
   npm run build
   node dist/server.cjs
   ```

### Этап 2: Установка приложения на Android

Приложение поддерживает устройства на базах Android 7.0 и новее (включая Android 14+).

---

#### а) Установка готового APK файла (Быстро)

1. Скачайте готовый скомпилированный APK-файл прямо из репозитория:
   👉 **[Скачать OpenMobileProxy.apk](https://github.com/makssest06-alt/open-mobile-proxy/raw/main/OpenMobileProxy.apk)**
2. Перенесите APK на смартфон и нажмите для установки (разрешите установку из сторонних источников в настройках безопасности Android при запросе).

---

#### б) Сборка приложения из исходных текстов

1. Откройте папку `android/` из репозитория в средах **Android Studio** или **IntelliJ IDEA**.
2. Убедитесь, что в системе установлен JDK 17 или JDK 21.
3. Выполните сборку APK через Gradle в терминале:
   ```bash
   cd android
   ./gradlew assembleDebug
   ```
   Скомпилированный файл будет находиться по пути `android/app/build/outputs/apk/debug/app-debug.apk`.

## 📲 Подключение телефона к серверу

После того как сервер запущен, а приложение установлено на смартфон, выполните привязку:

1. **Запустите приложение OpenMobileProxy** на смартфоне.
2. В поле **Server URL** введите адрес вашей панели управления, например:
   `http://217.114.8.131:3000`
3. В поле **Device ID** указано уникальное имя устройства (можно оставить по умолчанию или задать свое, например `phone_megafon`).
4. Нажмите зеленую кнопку **Запустить сервис** (`START PROXY SERVICE`).
5. В строке состояния появиться уведомление *"OpenMobileProxy is active"*.
6. *(Важно для автоматической смены IP)*:
   - При первом включении авто-ротации приложение попросит включить **Службу специальных возможностей (Accessibility Service)**.
   - Перейдите в *Настройки Android ➔ Специальные возможности ➔ Скачанные приложения ➔ OpenMobileProxy* и переключите тумблер в состояние **Включено**.

## 💻 Как пользоваться системой

### Использование SOCKS5 / HTTP прокси

После подключения устройства в веб-панели управления появится активная карточка прокси, например:
`admin:password@217.114.8.131:24759`

Вы можете скопировать её в один клик и использовать в любых программах, браузерах, антидетект-браузерах (Dolphin, AdsPower, Multilogin) или скриптах.

#### cURL
```bash
curl --socks5-hostname admin:password@217.114.8.131:24759 https://ifconfig.me
```

#### Python (requests / httpx)
```python
import requests

proxies = {
    'http': 'socks5h://admin:password@217.114.8.131:24759',
    'https': 'socks5h://admin:password@217.114.8.131:24759'
}

resp = requests.get('https://ifconfig.me', proxies=proxies)
print("Мобильный IP:", resp.text)
```

### Ротация IP (Смена адреса)

Смена IP-адреса выполняется путем мгновенного переключения авиарежима (Airplane Mode) на смартфоне:

1. **Из веб-панели**:
   - Нажмите кнопку **`🔄 Сменить IP`** напротив нужного устройства в таблице.
2. **По времени (Авто-ротация)**:
   - Включите тумблер в колонке **Авто-ротация** в таблице или в окне *Диагностика* и задайте интервал в минутах (например, каждые 5 минут). Сервер сам будет давать команду смартфону на смену IP.
3. **По API-ссылке (для скриптов)**:
   - Вы можете вызвать ротацию простым GET-запросом из любого стороннего софта:
     ```bash
     curl http://217.114.8.131:3000/api/change-ip/ИМЯ_УСТРОЙСТВА
     ```

### Панель управления и Диагностика

- **Тест скорости (SpeedTest)**: В окне *Диагностика* нажмите *Тест скорости*. Сервер проведет замер реальной скорости скачивания и отдачи мобильного прокси через соединение Cloudflare CDN.
- **Смена паролей**: В верхней панели кнопка **`🔒 Пароль`** позволяет сменить пароль от самой веб-панели.
- **Настройка доступа прокси**: Блок *Конфигурация сервера* позволяет изменить стандартные логин и пароль для SOCKS5 подключения или полностью отключить авторизацию.
